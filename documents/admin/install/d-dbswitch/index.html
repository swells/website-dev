Switching Databases for DeployR

Version 8.1.0
  • Introduction
    • Using SQL Server with DeployR
      • Using PostgreSQL with DeployR

        Introduction

        DeployR relies on an H2 database by default. However, you can use one of these databases instead:

        • On Windows, you can use local SQL Server database as a replacement for H2 locally, or use remote SQL Server database instead. You can also use SQL Server for high availability on your Windows machine.

        • On Linux, you can use local PostgreSQL database as a replacement for H2 locally, or use remote PostgreSQL database instead.

        After installing DeployR, but before using it, you can configure DeployR to use a different database.

        Using SQL Server with DeployR

        If you want to use a local or remote SQL Server database for DeployR instead of the default local H2 database, you'll need to:

        1. Install and configure SQL Server as described for that product.

        2. Download the JDBC driver for the version of the database you installed and copy them under both of the following folders:

          • $DEPLOYR_HOME\Apache_Tomcat\lib

          • $DEPLOYR_HOME\deployr\tools\lib

        3. Stop the DeployR server.

          1. Launch the DeployR administrator utility script:

            cd C:\Program Files\Microsoft\DeployR\8.1\deployr\ 
adminUtility.bat

          2. From the main menu, choose Option 1 to stop the DeployR Server.

        4. Update the database properties to point to the new database as follows:

          1. Open deployr.groovy, which is the DeployR server external configuration file.

          2. Locate the dataSource property block.

          3. Replace the contents of that block with these properties: 

            dataSource {
     dbCreate = "update"
     driverClassName = "com.microsoft.sqlserver.jdbc.SQLServerDriver"
     url = "jdbc:sqlserver://localhost;instanceName=SQLEXPRESS;database=TestDB2;integratedSecurity=true;"
}

            Note: If you are using a remote database, use the IP address or FQDN of the remote machine rather than localhost.

        5. If you are connecting to a remote SQL Server database, be sure to open the database port to the public IP of the DeployR server.

        6. Test the connection to the database.

          1. Launch the adminUtility script again.

          2. From the main menu, choose Option 5 to test the database connection. If there are any issues, you must solve them before continuing.

          3. Once the connection test passes, return the main menu.

          4. From the main menu, choose Option 1 to restart DeployR-related services.

          5. Once the DeployR server has been successfully restarted, return the main menu.

          6. From the main menu, choose Option 3 to run the full DeployR diagnostic tests. If there are any issues, you must solve them before continuing.

        Using PostgreSQL with DeployR

        If you want to use a local or remote PostgreSQL database for DeployR instead of the default local H2 database, you'll need to:

        1. Install and configure PostgreSQL as described for that product.

        2. Download the JDBC driver for the version of the database you installed and copy them under both of the following folders:

          • $DEPLOYR_HOME/tomcat/tomcat7/lib

          • $DEPLOYR_HOME/deployr/tools/lib

        3. Stop the DeployR server.

          1. Launch the DeployR administrator utility script:
            cd $DEPLOYR_HOME/deployr/tools/ 
./adminUtility.sh

        4. From the main menu, choose Option 1 to stop the DeployR Server.

        5. Update the database properties to point to the new database as follows:

          1. Open deployr.groovy, which is the DeployR server external configuration file.

          2. Locate the dataSource property block.

          3. Replace the contents of that block with these properties: 

              dataSource {
              dbCreate = "update"
              driverClassName = "org.postgresql.Driver"
              url = "jdbc:postgresql://localhost:5432/deployr"
              pooled = true
              username = "sa"
              properties {
              maxActive = -1
              minEvictableIdleTimeMillis=1800000
              timeBetweenEvictionRunsMillis=1800000
              numTestsPerEvictionRun=3
              testOnBorrow=true
              testWhileIdle=true
              testOnReturn=true
              validationQuery="SELECT 1"
              }
          }

            Note: If you are using a remote database, use the IP address or FQDN of the remote machine rather than localhost.

        6. If you are connecting to a remote PostgreSQL database, be sure to open the database port to the public IP of the DeployR server.

        7. Test the connection to the database.

          1. Launch the adminUtility script again.

          2. From the main menu, choose Option 5 to test the database connection. If there are any issues, you must solve them before continuing.

          3. Once the connection test passes, return the main menu.

          4. From the main menu, choose Option 1 to restart DeployR-related services.

          5. Once the DeployR server has been successfully restarted, return the main menu.

          6. From the main menu, choose Option 3 to run the full DeployR diagnostic tests. If there are any issues, you must solve them before continuing.
