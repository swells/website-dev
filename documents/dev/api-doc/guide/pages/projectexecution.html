6.2 Working with the Project Execution APIs

Callers can use the project execution APIs to facilitate the execution of R code and the retrieval of the project's execution history associated results.

R Code Execution

There are two ways to execute R code on a project:

  1. Execute a block of R code directly using the /r/project/execute/code call.
  2. Execute a repository-managed script using the /r/project/execute/script call.

Important! Any project execution can be interrupted using the /r/project/execute/interrupt call.

R Code Execution History

To retrieve the R code execution history on a project, use the /r/project/execute/history call.

R Code Execution Artifacts

Artifacts are files, that are explicitly named in R code and written to the project's working directory.

To retrieve an artifact on a project, use the /r/project/directory/download call.

R Code Execution Results

Results are files, that are implicitly named in R code, generated by the R graphics device and stored as part of a project's execution history.

To retrieve results on a project, use the /r/project/execute/result/download call.

6.2.1 /r/project/execute/code

This call executes a block of R code on the R session identified by the project parameter.

Please see the Standard Execution Model section in this guide for further details about the parameters on this call.

Some key data indicated in the response markup on this call:

  • code - indicates the code that has been executed
  • console - indicates the R console output resulting from the execution
  • results - indicates the list of files generated by the R graphics device
  • artifacts - indicates the list of files generated or modified in the working directory
  • objects - indicates the list of R objects returned from the workspace
  • files - indicates the list of files and objects stored in the repository after the execution completes
  • interrupted - indicates the interrupted status of execution
  • error - on failure, indicates the reason for failure
  • errorCode - on failure, indicates the error code for failure

HTTP Method

POST

Request Format

Content-Type: application/x-www-form-urlencoded; charset=utf-8

Parameters

  • format – specifies markup encoding on the call: json or xml
  • project – specifies the project identifier
  • code – specifies the block of R code

Pre-Execution Parameters

  • inputs - (optional) DeployR-encoded inputs
  • csvinputs - (optional) comma-separated list of primitive name/value inputs
  • preloadbydirectory - (optional) comma-separated list of repository directory names
  • preloadfilename - (optional) comma-separated list of repository filenames
  • preloadfiledirectory - (optional) comma-separated list of directories, one directory per preloadfilename
  • preloadfileauthor - (optional) comma-separated list of authors, one author per preloadfilename
  • preloadfileversion - (optional) comma-separated list of versions, one version per preloadfilename
  • preloadobjectname - (optional) comma-separated list of repository object (.rData) filenames
  • preloadobjectdirectory - (optional) comma-separated list of directories, one directory per preloadobjectname
  • preloadobjectauthor - (optional) comma-separated list of authors, one author per preloadobjectname
  • preloadobjectversion - (optional) comma-separated list of versions, one version per preloadobjectname
  • adoptworkspace – _(optional) identifies project whose workspace is to be adopted
  • adoptdirectory – _(optional) identifies project whose directory is to be adopted
  • adoptpackages – _(optional) identifies project whose package dependencies are to be adopted

On-Execution Parameters

  • phantom - (optional) if true the execution is treated as a phantom execution
  • tag - (optional) specifies a tag that labels the execution
  • echooff - (optional) if true R commands will not appear in the console output
  • consoleoff - (optional) if true console output is not returned in the response markup
  • artifactsoff - (optional) when enabled, artfiacts generated in the working directory are neither cached to the database nor reported in the response markup
  • enableConsoleEvents - (optional) if true console events are delivered on the event stream
  • graphics - (optional) specifies preferred R graphics device for execution: png or svg
  • graphicswidth - (optional) specifies preferred width for R graphics device images
  • graphicsheight - (optional) specifies preferred height for R graphics device images

Post-Execution Parameters

  • robjects – (optional) specifies a comma-separated list of objects for retrieval following the execution
  • storefile - (optional) comma-separated list of working directory filenames
  • storeobject - (optional) comma-separated list of workspace object names
  • storeworkspace - (optional) filename (.rData) where workspace contents will be saved in the repository
  • storedirectory - (optional) repository directory for stored files and objects after the execution completes, defaults to root
  • storenewversion - (optional) if true, ensures each file stored in repository results in new version being created if needed
  • storepublic - (optional) if true, publishes each file stored in the repository
  • infinity - (optional) specifies custom value for Infinity appearing in R object data returned on call, otherwise Infinity is represented by 0x7ff0000000000000L
  • nan - (optional) specifies custom value for NaN appearing in R object data returned on call, otherwise NaN is represented by null
  • encodeDataFramePrimitiveAsVector - (optional) if true, data.frame primitives are encoded vectors in R object data returned on call

6.2.2 /r/project/execute/script

This call executes repository-managed scripts or external scripts on the R session identified by the project parameter.

To execute a single repository-managed script the caller must provide parameter values for filename , author and optionally directory and version . To execute a chain of repository-managed scripts the caller must provide a comma-separated list of values on the filename , author and optionally directory and version parameters.

To execute a single external script the caller must provide a valid URL or file path using the externalsource parameter. To execute a chain of external scripts the caller must provide a comma-separated list of values on the externalsource parameter. Note, that in order to execute an external script, the caller must have been granted the POWER_USER role.

Note: A chained execution executes each of the scripts identified on the call in a sequential fashion on the R session, with execution occurring in the order specified on the parameter list.

Please see the Standard Execution Model section in this guide for further details about the parameters on this call.

Some key data indicated in the response markup on this call:

  • console - indicates the R console output resulting from the execution
  • results - indicates the list of files generated by the R graphics device
  • artifacts - indicates the list of files generated or modified in the working directory
  • objects - indicates the list of R objects returned from the workspace
  • files - indicates the list of files and objects stored in the repository after the execution completes
  • interrupted - indicates the interrupted status of execution
  • error - on failure, indicates the reason for failure
  • errorCode - on failure, indicates the error code for failure

POST

Content-Type: application/x-www-form-urlencoded; charset=utf-8

  • project – specifies the project identifier
  • filename – (optional) comma-separated list of repository-managed script filenames

  • author – (optional) comma-separated list of authors, one author per filename

6.2.5 /r/project/execute/history

This call retrieves the execution history for the specified project.

Some of the key data indicated for each history item in the response markup on this call include:

  • code - indicates the code that has been executed
  • console - indicates the R console output resulting from the code execution
  • resultsGenerated - indicates the number of generated results on the execution
  • resultsAvailable - indicates the number of generated results still stored on the execution
  • resourceUsage - indicates the current storage byte count for results on the execution
  • execution - indicates the execution identifier, which can then be used on /r/project/execution/result/download call to retrieve results
  • interrupted - indicates the interrupted status of execution
  • error - on failure, indicates the reason for failure
  • errorCode - on failure, indicates the error code for failure

HTTP Method

GET

Request Format

Content-Type: application/x-www-form-urlencoded; charset=utf-8

Parameters

  • format – specifies the markup encoding on the call: json or xml
  • project – specifies the project identifier
  • filterdepth - specifies the max number of executions to be returned in the history
  • execution - (optional) specifies the comma-separated list of execution identifiers on which to filter history
  • filtertag - (optional) specifies the execution tag on which to filter history
  • reversed - (optional) if true, the execution history is returned in a reverse-chronological order

Examples 

format=json
project=PROJECT-e2edd15f-dd67-42f6-8d82-90a2a5a669ca
filterdepth=100

6.2.6 /r/project/execute/flush

This call flushes executions in the history on the specified project.

Flushing an execution deletes both the R console output and the generated results associated with that execution but does not remove the execution itself from the history. By omitting the execution parameter, the caller can flush every execution in the history on the specified project.

This flushing facility is provided to help users manage the levels of resource usage associated with their persistent projects.

Content-Type: application/x-www-form-urlencoded; charset=utf-8

  • project – specifies the project identifier

6.2.8 /r/project/execute/result/download

This call downloads the execution results for the specified project.

By specifying a value for the execution parameter the caller can download only results on the specified executions. By specifying a value for the filename parameter the caller can download a specific result on the specified execution.

HTTP Method

GET

Request Format

Content-Type: application/x-www-form-urlencoded; charset=utf-8

Parameters

  • format – specifies markup encoding on the call: json or xml
  • project – specifies the project identifier
  • execution - (optional) specifies a comma-separated list of execution identifiers
  • filename - (optional) specifies a result file name
  • inline - (optional) if true, the Content-Disposition response header indicating attachment is omitted

Examples 

format=json
project=PROJECT-e2edd15f-dd67-42f6-8d82-90a2a5a669ca
execution=EXEC-76b503a3-3c2f-4114-b4b2-e3b1b6f88657

Response

If a value for the filename parameter is specified on the call, then:

Content-Type: result/content-type

If the filename parameter is omitted on the call, then:

Content-Type: application/zip
