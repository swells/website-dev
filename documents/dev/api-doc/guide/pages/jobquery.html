7.4 /r/job/query - Reference Documentation

Authors: Microsoft Corporation

Version: 8.0.0

7.4 /r/job/query

This call queries the job status. The status property will indicate one of the following values:

Scheduled, Queued, Running, Completed, Cancelling, Cancelled, Interrupted, Aborted, Failed.

HTTP Method

GET

Request Format

Content-Type: application/x-www-form-urlencoded; charset=utf-8

Parameters

  • format â€“ specifies markup encoding on the call: json or xml
  • job - specifies a comma-separated list of job identifiers
  • extended - (optional) if true, additional data properties describing each job are listed in the response markup

Examples 

format=json
job=JOB-91baaca8-574d-4703-b225-97e64ec63e70

{
    "deployr": {
        "response": {
            "call": "/r/job/query",
            "success": true,
            "job" : {
                "job": "JOB-91baaca8-574d-4703-b225-97e64ec63e70",
                "name": "Scheduled Analysis",
                "descr": "Demonstrates job scheduling."
                "schedstart": 1316751060000,
                "schedrepeat": 2,
                "schedinterval": 86400000,
